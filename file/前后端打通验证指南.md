# 🔗 前后端打通验证指南

## 📋 验证步骤

### 第一步：启动后端服务

#### 1. 确保环境配置正确

```bash
# 检查 Java 版本
java -version
# 应该显示 Java 17+ 或 Java 21

# 检查 Maven 版本
mvn -version
# 应该显示 Maven 3.6+

# 检查 MySQL 版本
mysql --version
# 应该显示 MySQL 5.7+ 或 8.0+
```

#### 2. 初始化数据库

```bash
# 进入项目目录
cd f:\program\graduation project\lvxingyouyang\back-end

# 创建数据库并导入初始化脚本
mysql -u root -p < init.sql

# 或者手动执行
mysql -u root -p
mysql> CREATE DATABASE lvxingyouyang;
mysql> USE lvxingyouyang;
mysql> SOURCE init.sql;
```

#### 3. 修改数据库配置

编辑 `src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/lvxingyouyang?useSSL=false&serverTimezone=UTC&allowPublicKeyRetrieval=true&characterEncoding=utf8mb4
    username: root
    password: your_password  # 改为你的 MySQL 密码
    driver-class-name: com.mysql.cj.jdbc.Driver
  
  jpa:
    hibernate:
      ddl-auto: update
    properties:
      hibernate:
        dialect: org.hibernate.dialect.MySQL8Dialect
        format_sql: true

server:
  port: 8080
  servlet:
    context-path: /api

jwt:
  secret: your-secret-key-here-at-least-32-characters-long
  expiration: 86400000  # 24 小时
```

#### 4. 编译并运行后端

```bash
# 编译项目
mvn clean install

# 运行应用
mvn spring-boot:run

# 或者直接运行 JAR
java -jar target/lvxingyouyang-backend-1.0.0.jar
```

#### 5. 验证后端启动成功

```
应该看到类似的输出：
Started LvxingyouyangApplication in 5.123 seconds (JVM running for 5.456)
```

### 第三步：前端连接配置

#### 1. 修改前端 API 基础 URL

在前端项目中，找到 API 配置文件（通常是 `src/api/config.js` 或 `src/config/api.js`）：

```javascript
// 修改为后端地址
const API_BASE_URL = 'http://localhost:8080/api';

export default API_BASE_URL;
```

#### 2. 配置 CORS（跨域）

后端已配置 CORS，支持以下来源：
- `http://localhost:3000` (React)
- `http://localhost:5173` (Vite)

如需修改，编辑 `SecurityConfig.java`：

```java
configuration.setAllowedOrigins(Arrays.asList(
    "http://localhost:3000",
    "http://localhost:5173",
    "http://localhost:8081"  // 添加你的前端地址
));
```

#### 3. 启动前端服务

```bash
# 进入前端项目目录
cd f:\program\graduation project\lvxingyouyang\front-end

# 安装依赖
npm install

# 启动开发服务器
npm run dev
# 或
npm start
```

### 第五步：调试和排查

#### 常见问题

**1. 后端无法启动**

```bash
# 检查端口是否被占用
netstat -ano | findstr :8080

# 检查数据库连接
# 确保 MySQL 正在运行
# 确保数据库名称和密码正确
```

**2. 前端无法连接后端**

```bash
# 检查后端是否运行
curl http://localhost:8080/api/hotels/popular

# 检查 CORS 配置
# 检查前端 API 基础 URL 是否正确
# 检查浏览器控制台错误信息
```

**3. 数据库连接失败**

```bash
# 检查 MySQL 是否运行
mysql -u root -p

# 检查数据库是否存在
SHOW DATABASES;

# 检查表是否创建
USE lvxingyouyang;
SHOW TABLES;
```

#### 查看日志

**后端日志：**
```bash
# 在控制台查看实时日志
# 或查看日志文件
tail -f logs/application.log
```

**前端日志：**
```bash
# 打开浏览器开发者工具
# F12 或 右键 -> 检查 -> Console 标签
```
