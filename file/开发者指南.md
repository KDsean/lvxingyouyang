## 🔑 核心概念

### 分层架构
```
Request
   ↓
Controller (接收请求)
   ↓
Service (业务逻辑)
   ↓
Repository (数据访问)
   ↓
Database (数据存储)
```

### 认证流程
```
1. 用户登录 → POST /auth/login
2. 获取 Token → 返回 JWT Token
3. 请求时添加 Token → Authorization: Bearer <token>
4. 过滤器验证 Token → JwtAuthenticationFilter
5. 允许访问 → 执行业务逻辑
```

## 💡 常见任务

### 添加新的 API 端点

#### 1. 创建 Entity
```java
@Entity
@Table(name = "my_table")
public class MyEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    // ... 字段
}
```

#### 2. 创建 Repository
```java
@Repository
public interface MyRepository extends JpaRepository<MyEntity, Long> {
    // 自定义查询方法
}
```

#### 3. 创建 Service
```java
@Service
@RequiredArgsConstructor
public class MyService {
    private final MyRepository repository;
    
    public MyEntity getById(Long id) {
        return repository.findById(id)
            .orElseThrow(() -> new RuntimeException("Not found"));
    }
}
```

#### 4. 创建 Controller
```java
@RestController
@RequestMapping("/my-endpoint")
@RequiredArgsConstructor
public class MyController {
    private final MyService service;
    
    @GetMapping("/{id}")
    public ApiResponse<?> getById(@PathVariable Long id) {
        return ApiResponse.success(service.getById(id));
    }
}
```

```sql
CREATE TABLE my_table (

);
```

#### 2. 创建对应的 Entity 类
#### 3. 创建对应的 Repository 接口

### 修改 API 响应格式

所有 API 响应都使用统一格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

使用 `ApiResponse` 类：
```java
// 成功响应
return ApiResponse.success(data);

// 错误响应
return ApiResponse.error(400, "错误信息");
```

## 🔐 认证和授权

### 获取 Token
```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"user","password":"pass"}'
```

### 使用 Token
```bash
curl -X GET http://localhost:8080/api/auth/user \
  -H "Authorization: Bearer <token>"
```

### 添加认证保护
在 `SecurityConfig.java` 中配置：
```java
.authorizeHttpRequests(authz -> authz
    .requestMatchers("/public/**").permitAll()
    .anyRequest().authenticated()
)
```

## 🚨 常见问题

### Q: 数据库连接失败
**A**: 检查 MySQL 是否运行，修改 `application.yml` 中的连接信息

### Q: 端口 8080 已被占用
**A**: 在 `application.yml` 中修改端口：
```yaml
server:
  port: 8081
```

### Q: Token 过期
**A**: 在 `application.yml` 中修改过期时间：
```yaml
jwt:
  expiration: 86400000  # 24 小时
```

### Q: CORS 错误
**A**: 检查 `SecurityConfig.java` 中的 CORS 配置

### Q: 权限不足
**A**: 检查是否添加了 Authorization 请求头
