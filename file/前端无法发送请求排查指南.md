## 🔍 前端无法发送请求的常见原因

### 原因 1：CORS 错误（最常见）

**症状：**
```
Access to XMLHttpRequest at 'http://localhost:8080/api/...' 
from origin 'http://localhost:3000' has been blocked by CORS policy
```

**解决方案：**
1. ✅ 已修复 - 后端 CORS 配置已更新
2. 重新编译后端：`mvn clean install`
3. 重启后端服务

### 原因 2：后端未启动

**症状：**
```
Failed to fetch
Network error
```

**解决方案：**
```bash
# 检查后端是否运行
curl http://localhost:8080/api/hotels/popular

# 如果无响应，启动后端
cd back-end
mvn spring-boot:run
```

### 原因 3：API 地址错误

**症状：**
```
404 Not Found
```

**解决方案：**
1. 检查 `.env.development` 中的 `VITE_API_BASE_URL`
2. 应该是：`http://localhost:8080/api`
3. 不应该是：`http://localhost:8080` 或 `http://localhost:8080/`

### 原因 4：前端环境变量未加载

**症状：**
```
请求发送到错误的地址
```

**解决方案：**
```bash
# 重启前端开发服务器
cd front-end
npm run dev
```

## 🚀 完整的排查步骤

### 步骤 1：验证后端是否运行

```bash
# 在终端中执行
curl http://localhost:8080/api/hotels/popular

# 或在浏览器中访问
http://localhost:8080/api/hotels/popular
```

**预期结果：**
```json
{
  "code": 200,
  "message": "success",
  "data": [...]
}
```

**如果无响应：**
```bash
# 启动后端
cd back-end
mvn spring-boot:run
```

### 步骤 2：验证前端环境变量

**文件：** `front-end/.env.development`

```env
VITE_API_BASE_URL=http://localhost:8080/api
VITE_AI_API_URL=http://localhost:8080/ai
```

**验证方法：**
1. 打开浏览器开发者工具（F12）
2. 在 Console 中执行：
```javascript
console.log(import.meta.env.VITE_API_BASE_URL)
// 应该输出：http://localhost:8080/api
```

### 步骤 3：验证 CORS 配置

**后端配置：** `SecurityConfig.java`

```java
configuration.setAllowedOrigins(Arrays.asList(
    "http://localhost:3000",
    "http://localhost:5173",
    "http://127.0.0.1:3000",
    "http://127.0.0.1:5173"
));
```

**验证方法：**
1. 重新编译后端：`mvn clean install`
2. 重启后端服务
3. 在浏览器中检查 Network 标签

### 步骤 4：检查浏览器网络请求

**操作步骤：**
1. 打开浏览器开发者工具（F12）
2. 点击 **Network** 标签
3. 执行一个 API 请求（如登录）
4. 查看请求详情

**检查项：**
- ✅ 请求 URL 是否正确
- ✅ 请求方法是否正确（GET/POST）
- ✅ 响应状态码是否是 200
- ✅ 响应头中是否有 CORS 相关信息

### 步骤 5：检查浏览器控制台错误

**操作步骤：**
1. 打开浏览器开发者工具（F12）
2. 点击 **Console** 标签
3. 查看是否有错误信息

**常见错误：**

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| CORS policy error | CORS 配置不正确 | 检查后端 CORS 配置 |
| 404 Not Found | API 地址错误 | 检查 API 地址 |
| Failed to fetch | 后端未运行 | 启动后端 |
| 401 Unauthorized | Token 无效 | 重新登录 |

## 🚨 如果仍然无法连接

### 检查清单

```bash
# 1. 检查后端是否运行
netstat -ano | findstr :8080

# 2. 检查前端是否运行
netstat -ano | findstr :3000
netstat -ano | findstr :5173

# 3. 检查 MySQL 是否运行
mysql -u root -p

# 4. 检查防火墙
# Windows：检查防火墙是否阻止了端口
# Mac/Linux：检查 iptables 规则
```

### 重置所有配置

```bash
# 1. 停止所有服务（Ctrl+C）

# 2. 清理后端
cd back-end
mvn clean

# 3. 清理前端
cd front-end
rm -rf node_modules
npm install

# 4. 重新启动
# 终端 1
cd back-end
mvn spring-boot:run

# 终端 2
cd front-end
npm run dev
```
