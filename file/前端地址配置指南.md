## 2️⃣ 后端 CORS 配置（添加前端地址）

### 位置：后端 `SecurityConfig.java`

**文件路径：** `f:\program\graduation project\lvxingyouyang\back-end\src\main\java\com\lvxingyouyang\config\SecurityConfig.java`

### 当前配置：

```java

    CorsConfiguration configuration = new CorsConfiguration();
    // 允许的来源
    configuration.setAllowedOrigins(Arrays.asList(
        "http://localhost:3000",      // React 默认端口
        "http://localhost:5173"       // Vite 默认端口
    ));
    // ... 其他配置
}
```

### 需要添加的前端地址

根据你的前端启动方式，添加相应的地址：

#### 方案 1：Vite 开发服务器（推荐）

```java
configuration.setAllowedOrigins(Arrays.asList(
    "http://localhost:3000",      // React
    "http://localhost:5173",      // Vite (Vue)
    "http://localhost:8081"       // 其他可能的端口
));
```

#### 方案 2：生产环境

```java
configuration.setAllowedOrigins(Arrays.asList(
    "http://localhost:3000",
    "http://localhost:5173",
    "https://your-domain.com",    // 你的生产域名

));
```

#### 方案 3：允许所有来源（仅开发环境）

```java

```

## 🔍 调试技巧

### 1. 检查 API 请求

打开浏览器开发者工具（F12）：
- 点击 **Network** 标签
- 执行一个 API 请求（如登录）
- 查看请求 URL 是否正确
- 查看响应状态码（应该是 200）

### 2. 检查 CORS 错误

如果看到 CORS 错误：
```
Access to XMLHttpRequest at 'http://localhost:8080/api/...' 
from origin 'http://localhost:3000' has been blocked by CORS policy
```

**解决方案：**
1. 确认后端 CORS 配置中包含前端地址
2. 确认后端已重新编译和启动
3. 清除浏览器缓存

### 3. 检查 Token 认证

如果看到 401 错误：
```
Authorization header missing or invalid
```

**解决方案：**
1. 确认已成功登录
2. 确认 Token 已保存到 localStorage
3. 检查 `request.ts` 中的 Token 添加逻辑
