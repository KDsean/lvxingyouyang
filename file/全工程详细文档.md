## 1. 项目概览

“旅行有样”是一个旅游服务综合平台，核心能力包括：

- 用户注册、登录、个人资料维护。
- 酒店、机票、火车票、租车、景点、美食、购物等旅游资源查询。
- 目的地浏览、目的地详情、热门目的地和目的地热力图。
- 当地攻略列表、详情、发布、点赞和搜索。
- 用户行程计划、历史规划、订单管理。
- AI 行程规划，对接 Python FastAPI + LangChain/MCP 服务。

工程采用三段式架构：

- `front-end/`：Vue 3 + Vite + TypeScript 前端。
- `back-end/`：Spring Boot 3.2 + MySQL + JPA + Spring Security/JWT 后端。
- `agent/`：Python FastAPI + LangChain + MCP 的 AI 旅行规划服务。

典型调用链：

```text
浏览器 Vue 前端
  -> /api/... 通过 Vite 代理或生产环境网关访问 Spring Boot
  -> Spring Boot /api 上下文路径处理业务接口
  -> MySQL 保存用户、资源、行程、订单等数据

AI 对话链路：
浏览器
  -> /api/ai/chat 或 /api/ai/chat/stream
  -> Spring Boot AIController
  -> Python FastAPI http://localhost:8001/chat 或 /chat/stream
  -> LangChain / MCP 工具 / 大模型
```

## 5. 后端结构

```text
back-end/src/main/java/com/lvxingyouyang/
├─ LvxingyouyangApplication.java   应用启动类
├─ config/                         全局配置
├─ controller/                     REST 控制器
├─ dto/                            请求/响应 DTO
├─ entity/                         JPA 实体
├─ repository/                     Spring Data JPA 仓库
├─ security/                       JWT 认证
├─ service/                        服务接口
├─ service/impl/                   服务实现
└─ util/                           工具类
```

### 5.1 配置层

- `AppConfig`
  - 提供通用 Bean，例如 `RestTemplate`。
- `SecurityConfig`
  - 禁用 CSRF。
  - 启用 CORS。
  - 使用无状态 Session。
  - 放行 `/auth/login`、`/auth/register`。
  - 放行酒店、机票、火车、租车、景点、攻略、美食、商品、目的地、AI 等查询接口。
  - 其他请求需要 JWT 认证。
  - 注册 `JwtAuthenticationFilter`。
- `WebConfig`
  - Web MVC 相关配置。
- `GlobalExceptionHandler`
  - 全局异常处理，统一包装错误响应。

### 5.2 安全认证

- `JwtTokenProvider`
  - 使用 `jwt.secret` 签名。
  - Token subject 存储用户 ID。
  - 默认 24 小时过期。
  - 提供生成、解析、校验 Token 方法。
- `JwtAuthenticationFilter`
  - 从 `Authorization: Bearer <token>` 读取 JWT。
  - 校验成功后把用户 ID 放入 Spring Security 的 `Authentication.principal`。

前端会在 Axios 请求拦截器中读取 `localStorage.token` 并添加：

```http
Authorization: Bearer <token>
```

### 5.3 统一响应

后端统一使用：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

主要 DTO：

- `ApiResponse<T>`：统一响应。
- `PageResponse<T>`：分页响应，包含 `list`、`total`、`page`、`pageSize`。
- `LoginRequest`：登录请求，字段 `username`、`password`。
- `RegisterRequest`：注册请求，字段 `username`、`email`、`password`、`phone`。
- `LoginResponse`：登录成功后返回 `token` 和 `user`。
- `UserDTO`：用户公开信息，不包含密码。
- `DestinationHeatmapPoint`：目的地热力图点位。

### 6.1 认证模块 `/auth`

控制器：`AuthController`  
服务：`IAuthService`、`AuthServiceImpl`  
实体：`User`

接口：

| 方法 | 路径 | 说明 | 认证 |
|---|---|---|---|
| POST | `/auth/register` | 用户注册，返回 token 和用户信息 | 否 |
| POST | `/auth/login` | 用户登录，支持用户名/邮箱/手机号作为登录标识 | 否 |
| GET | `/auth/user` | 获取当前登录用户 | 是 |
| PUT | `/auth/profile` | 更新当前用户资料 | 是 |
| POST | `/auth/logout` | 登出，后端返回成功消息 | 通常需要 |

业务要点：

- 注册时校验用户名和邮箱唯一。
- 密码使用 BCrypt 加密。
- 登录成功后签发 JWT。
- 更新资料时会避免用户名/邮箱与其他用户冲突。

## 8. 数据库脚本

位置：

```text
back-end/src/main/resources/
├─ application.yml
├─ data.sql
├─ init.sql
├─ init1.sql
└─ mock_data.sql
```

脚本用途：

- `init.sql`：较完整的建表脚本，包含用户、目的地、景点、航班、酒店、火车、租车、餐厅、攻略、商品等表和集合表。
- `init1.sql`：另一份建表脚本，包含类似表结构，同时包含 `orders`。
- `mock_data.sql`：模拟数据插入脚本，覆盖用户、目的地、景点、酒店、航班、火车、租车、餐厅、攻略、商品、行程等。
- `data.sql`：当前 Spring Boot 配置会自动加载的初始化数据脚本。

当前 `application.yml` 中：

```yaml
spring:
  sql:
    init:
      mode: always
      data-locations: classpath:data.sql
  jpa:
    hibernate:
      ddl-auto: update
    defer-datasource-initialization: true
```

含义：

- Hibernate 先根据实体更新表结构。
- 再执行 `data.sql`。
- 每次启动都会尝试执行初始化 SQL，脚本需要具备幂等性，否则可能重复插入或报唯一键错误。

## 10. AI Agent 详细说明

### 10.1 FastAPI 服务

文件：`agent/server.py`

接口：

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/health` | 返回 `{"status":"ok","service":"travel-agent"}` |
| POST | `/chat` | 非流式对话，返回 `content` 和 `sessionId` |
| POST | `/chat/stream` | SSE 流式对话，返回 `data: {"content":"..."}` 和 `data: [DONE]` |

请求模型：

```json
{
  "message": "帮我规划北京到上海三日游",
  "history": [
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."}
  ],
  "sessionId": "optional-session-id"
}
```

响应模型：

```json
{
  "content": "AI 回复内容",
  "sessionId": "optional-session-id"
}
```

### 10.2 TravelAgent 核心

文件：`agent/travel_agent.py`

核心职责：

- 读取环境变量：
  - `OPENAI_API_KEY`
  - `OPENAI_BASE_URL`
  - `MODEL_NAME`
  - `AMAP_API_KEY`
  - `VARIFLIGHT_API_KEY`
  - `MCP_12306_URL`
- 构建 OpenAI 兼容模型客户端。
- 构建 MCP 服务配置。
- 尝试加载：
  - 高德地图 MCP
  - 飞常准 MCP
  - 12306 MCP
- 判断用户问题是否为旅行相关。
- 对复杂旅行问题使用 ReAct Agent 和工具。
- 对普通文本问题直接调用 LLM。
- 支持普通输出和流式输出。
- 对火车、航班、城市、机场等旅行参数做解析和工具查询。
- 关闭时释放 MCP 客户端资源。

### 10.3 MCP 工具

设计上支持：

- 高德地图：POI、路线、天气、地理编码等。

  - 前端有 `/ai/generate-plan`、`/ai/optimize-plan`
  - 当前后端和 Python 服务主要提供 `/chat` 与 `/chat/stream`。

### 12.4 占位接口

以下接口当前主要返回成功字符串，并未创建真实订单或支付记录：

- 酒店预订：`POST /hotels/book`
- 机票预订：`POST /flights/book`
- 火车预订：`POST /trains/book`
- 租车预订：`POST /cars/book`
- 景点购票：`POST /attractions/buy-ticket`
- 餐厅预订：`POST /restaurants/book`
- 商品购物车/下单：部分为占位结果

如果用于毕业设计演示，前端可展示成功状态；如果用于真实闭环，需要补充订单落库、库存/余票扣减、支付状态、用户归属校验。

### 12.5 权限归属校验

当前行程和订单部分接口虽然要求登录，但详情、更新、删除、取消等操作需要进一步确认是否校验资源归属。例如：

- 用户 A 不应能访问用户 B 的订单详情。
- 用户 A 不应能更新或删除用户 B 的行程。

建议服务层按 `id + userId` 查询，或在取出实体后比较 `userId`。

### 12.6 AIController JSON 序列化

`AIController.toJson(Map<String,Object>)` 是手写序列化，对嵌套数组和对象不稳。AI 请求里常有 `history` 列表，建议替换为：

```java
private final ObjectMapper objectMapper;
String jsonBody = objectMapper.writeValueAsString(body);
```

### 12.7 数据初始化

`spring.sql.init.mode: always` 会在每次启动执行 `data.sql`。如果脚本非幂等，可能导致重复数据或启动失败。建议：

- 开发环境可保留 `always`。
- 演示或生产环境改为 `never` 或使用 Flyway/Liquibase。
- 插入脚本使用 `INSERT IGNORE` 或唯一键判断。

## 14. 常见排查

### 14.1 前端请求 404

检查：

- 前端 API 路径是否与后端 Controller 一致。
- Vite 代理 `/api` 是否指向 `http://localhost:8080`。
- 后端是否使用 `/api` context-path。

### 14.2 前端请求 401

检查：

- `localStorage.token` 是否存在。
- Header 是否为 `Authorization: Bearer <token>`。
- JWT 是否过期。
- 后端接口是否被 `SecurityConfig` 要求认证。

### 14.3 后端启动失败

检查：

- MySQL 是否启动。
- 数据库 `lvxingyouyang` 是否存在。
- `application.yml` 用户名密码是否正确。
- `data.sql` 是否重复插入唯一字段。
- Java 版本是否为 17。

### 14.4 AI 不回复

检查：

- Python 服务是否启动在 `8001`。
- `GET http://localhost:8001/health` 是否正常。
- `agent/.env` 是否配置 `OPENAI_API_KEY`。
- `OPENAI_BASE_URL` 和 `MODEL_NAME` 是否匹配供应商。
- 如使用 MCP，Node.js/npm 是否可用，外部 API Key 是否正确。

### 14.5 SSE 流式乱码或不显示

检查：

- Python 返回是否为 `text/event-stream`。
- 前端是否按 `data:` 行解析。
- 是否存在 Spring Boot 二次包装 SSE 的问题。
- 需要时使用 `front-end/src/api/modules/planning.ts` 的直连 Python 流式方案。
